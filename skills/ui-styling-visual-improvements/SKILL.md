# UI Styling & Visual Improvements

## Objetivo
Padronizar melhorias visuais no frontend com foco em clareza, consistencia e usabilidade para usuario leigo.

## Quando usar
- Ajustes de layout, espacos, tipografia e hierarquia visual.
- Refino de formularios, tabelas, cards e estados de erro/sucesso.
- Melhorias de legibilidade em dashboard operacional e financeiro.

## Fluxo rapido
1. Avaliar tela atual (densidade, contraste, alinhamento, responsividade).
2. Aplicar padroes base de UI (tokens, grid, escalas, componentes).
3. Melhorar experiencia de formulario (labels, dicas, validacao visual, feedback).
4. Melhorar estados da interface (loading, vazio, erro, sucesso).
5. Validar em desktop e mobile e registrar checklist final.

## Padroes obrigatorios
- Texto legivel (contraste alto e tamanho minimo adequado).
- Espacamento consistente (escala 4/8/12/16/24).
- Botoes com acao clara e estados hover/disabled.
- Formularios com mensagens amigaveis e erro tecnico opcional em dev.
- Nao quebrar fluxo existente de negocio (lote, lancamentos, KPIs).

## Entregaveis esperados
- CSS/estilos revisados com consistencia visual.
- Componentes principais com estados completos.
- Checklist de validacao visual concluido.

## Guardrails
- Evitar refatoracao estrutural grande sem necessidade.
- Evitar mudancas que alterem contratos de API.
- Priorizar simplicidade para produtor com ate 30 cabecas.
